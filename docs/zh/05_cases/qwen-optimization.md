# Qwen 系列模型训推与算子优化

这是 hello-cann 第一阶段主线案例。

## 目标

- 在昇腾 NPU 上完成 Qwen 小模型推理部署，并按机器条件记录更大模型结果。
- 基于同一模型完成单卡 LoRA 微调和推理验证。
- 使用 profiling 找到推理链路中的热点或可融合位置。
- 选择一个 LLM 常用算子，完成 Ascend C 实现和单独测试。
- 将自定义算子接入 PyTorch / `torch_npu` 推理流程，比较优化前后的性能和正确性。

## 章节串法

1. 环境章节：检查 NPU、CANN、`torch_npu`，记录版本。
2. 推理章节：跑通 Qwen 小模型，再按机器条件记录更大模型结果。
3. 微调章节：完成单卡 LoRA 微调，验证模型输出变化。
4. profiling 章节：定位推理热点，解释相关组件在问题链路中的位置。
5. 算子章节：从 PyTorch 参考实现开始，完成一个 CANN 版本。
6. 模型接入：把算子接回 Qwen 链路，做正确性和性能对比。
7. 应用接入：把 Qwen 服务接到一个上层应用，记录任务总耗时。
8. 总结章节：整理性能记录、代码结构、问题清单和复现实验表。

## 应用接入位置

Qwen 服务跑稳后，可以作为 `hello-agent` 或 `hello-claw` 的后端模型。这里不要求公开 API，内网部署的 OpenAI 兼容接口也可以。案例报告里记录服务地址格式、模型名、streaming 是否可用、首 token 延迟和应用任务耗时。
