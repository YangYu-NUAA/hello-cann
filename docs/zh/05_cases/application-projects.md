# 昇腾模型服务接入应用项目

这一页记录如何把部署在昇腾上的模型服务接到上层应用。

这里的模型服务可以来自本课程部署的 Qwen，也可以来自单位内网已有的大模型服务。应用侧只需要拿到 OpenAI 兼容接口，通常包括 `base_url`、`api_key` 和 `model`。

## 最小接入信息

| 项目 | 说明 |
|:---|:---|
| 服务地址 | OpenAI 兼容接口地址 |
| 模型名称 | 应用侧传入的 `model` |
| streaming | 是否支持流式输出 |
| 超时设置 | 单次请求超时时间 |
| 并发设置 | 并发 1 / 4 / 8 的测试结果 |
| 日志 | 请求、响应、错误和耗时 |

## 应用项目

| 项目 | 适合验证什么 |
|:---|:---|
| `hello-agent` | agent 任务能否稳定调用昇腾模型服务，记录模型调用次数和任务总耗时 |
| `hello-claw` | OpenClaw / Skill 类应用如何切换到昇腾模型服务 |
| `torch-rechub` / `rechub` | 推荐模型在 CANN 上的训练、推理和 profiling；后续可接 LLM 解释或运营文案生成 |

## 记录指标

| 指标 | 说明 |
|:---|:---|
| 首 token 延迟 | streaming 场景下用户第一次看到输出的时间 |
| 单次响应耗时 | 一次普通问答或工具调用的总耗时 |
| 应用任务耗时 | agent 完成整个任务的总时间 |
| 模型调用次数 | 一次任务中请求模型的次数 |
| 成功率 | 固定并发下成功完成的比例 |
| 错误类型 | 超时、上下文过长、服务断开、格式错误等 |

## 第一版做法

先用本课程部署的 Qwen 服务接一个最小应用，再补真实应用场景。

1. 启动 Qwen 推理服务。
2. 用 OpenAI 兼容接口发送一条普通请求。
3. 接入 `hello-agent` 或 `hello-claw` 的模型配置。
4. 固定一个小任务，记录请求日志和耗时。
5. 改并发或输出长度，再记录一次。
6. 把服务配置、命令、指标和问题写进案例报告。
