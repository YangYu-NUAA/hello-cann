# 社区任务描述：hello-cann 开源课程共建

## 任务名称

hello-cann：面向昇腾 CANN 的开源实战课程建设

## 任务背景

Datawhale 计划与华为昇腾团队共建 hello-cann 开源教学项目。这个项目面向希望上手昇腾 NPU、CANN 软件栈和 Ascend C 算子开发的学习者，内容从环境配置、模型训推、性能分析到算子开发逐步展开。

课程会从可用的昇腾机器出发，先解决环境和版本问题，再跑通模型推理、微调、性能分析和自定义算子开发，最后用 1-2 个案例把这些内容串起来。项目侧已有较充足的昇腾算力，因此案例可以从最小示例走到中等规模模型，并保留更接近真实业务的优化记录。

## 课程简介

hello-cann 是一门面向昇腾 CANN 的开源实战课程。课程从环境检查开始，逐步进入大模型推理、LoRA 微调、profiling、瓶颈分析、Ascend C 自定义算子和模型接入。

课程以模型训推和优化实验为主线。`torch_npu` 放在环境和推理章节，ACL、ATC/OM 放在最小推理和模型转换章节，GE 放在图编译和算子接入问题中，HCCL 放在通信概念和常见报错索引中。

每篇教程都要写清楚硬件型号、系统版本、CANN 版本、框架版本、运行命令、关键输出和验证方式。

## 课程目标

- 帮助学习者跑通昇腾环境、CANN 工具链和 PyTorch NPU 开发流程。
- 让有 PyTorch 或大模型基础的开发者理解 `torch_npu`、CANN、ACL/ATC、GE、HCCL、Ascend C、profiling 在模型训推和算子接入中的位置。
- 通过贯穿案例训练工程能力：会部署模型、会记录指标、会定位瓶颈、会判断是否需要自定义算子。
- 沉淀可复现的教程、代码、FAQ、性能记录和优化报告。
- 形成一批能展示学习成果的材料，例如 PR、算子实现、profile 报告、优化前后对比表。

## 适合对象

- 有 Python、PyTorch 或大模型应用基础，希望上手昇腾 NPU 的开发者。
- 想系统学习 CANN、`torch_npu`、Ascend C、自定义算子和模型优化的学生、工程师和研究人员。
- 负责模型迁移、推理服务、训练优化、算子开发或性能调优的工程开发者。
- 对算子开发、算子竞赛、开源算子贡献感兴趣的社区开发者。

## 课程大纲

### 0. 环境与版本基线

确认硬件、驱动、固件、CANN、Python、PyTorch、`torch_npu` 和常用工具是否匹配。输出环境检查清单、版本矩阵和常见问题记录。

### 1. CANN 软件栈与第一次上手

提供一张软件栈关系图，讲清楚 NPU、AI Core、CANN、`torch_npu`、ACL、ATC/OM、GE、HCCL、Ascend C 分别出现在什么环节。通过 NPU Hello World、算子 API 调用或最小推理程序完成第一次上手。

### 2. 大模型推理部署

基于 `torch_npu`、Transformers、vLLM-Ascend 或 MindIE 跑通 Qwen 系列模型推理，启动 OpenAI 兼容服务，记录显存占用、吞吐、首 token 延迟等基础指标。

### 3. 微调与训练实践

以 Qwen 等开源模型为主线，完成 LoRA 微调、训练日志记录、权重保存和推理验证。多卡训练与 HCCL 放在概念说明和排障材料里，实操仍按单卡环境设计。

### 4. Profiling 与瓶颈定位

学习使用 PyTorch profiling、CANN profiling、msProf 等工具观察模型执行过程，定位热点算子、Host 下发、数据搬运和显存瓶颈。

### 5. Ascend C 自定义算子开发

从 Add / Vector 算子开始，逐步学习 kernel 编写、tiling、数据搬运、工程化算子开发、pybind/aclnn 调用、调试和性能优化。

### 6. 算子接入模型与效果验证

把自定义算子接入 PyTorch 模型或推理链路，对比优化前后的耗时、显存、吞吐和正确性。这一节要说明为什么写、怎么接入、有没有真的变好。

### 7. 综合案例与复盘

通过 1-2 个贯穿案例，把环境、推理、微调、profiling、算子开发、模型接入和性能记录串起来，形成可以展示的项目成果。

## 课程需要覆盖的内容

课程内容围绕三件事展开。

- 跑通环境和模型：CANN 环境检查、`torch_npu` 验证、NPU Hello World、模型推理和最小微调。

- 看懂性能问题：benchmark 模板、profiling 工具、热点算子定位、Host 和 Device 时间拆分、显存、数据搬运和运行链路排查。

- 做一次真实修改：Ascend C 基础算子、tiling、数据搬运、融合算子、算子接入 PyTorch/`torch_npu`、正确性验证和性能对比。

## 组件出现位置

- `torch_npu`：放在环境验证、模型推理和训练章节，是 PyTorch 读者进入昇腾的主要入口。
- ACL / AscendCL：放在软件栈图和最小推理示例里，帮助读者理解运行时、内存、stream、模型执行这些概念。
- ATC / OM：放在推理部署和 FAQ 里，用来解释模型转换、离线模型和图编译报错。
- GE / GraphEngine：放在图编译和图优化背景里，知道它和图准备、图切分、图编译、图执行有关即可，不做源码讲解。
- HCCL：放在软件栈关系、通信概念和常见错误里，说明它在多卡场景中的位置。
- `cann-ops-adv`：作为融合算子参考，适合在 Ascend C 章节后给读者继续阅读。
- MindSpeed / MindSpeed-LLM：作为大模型训练和优化的进阶材料，主线案例验证到再展开。

## 配套案例规划

配套案例先做 1 个主线案例，另留 1 个可选拓展案例。主线案例贯穿课程，可选案例展示 CANN 在其他模型类型上的迁移能力。

### 案例一：Qwen 系列模型训推与算子优化

这是主线案例。

案例目标：

- 在昇腾 NPU 上完成 Qwen 小模型到中等规模模型的推理部署。
- 基于同一模型完成 LoRA 微调和推理验证。
- 使用 profiling 找到推理链路中的热点或可融合位置。
- 选择一个适合教学的算子点，完成 Ascend C 自定义算子开发。
- 将自定义算子接入 PyTorch / `torch_npu` 推理流程，比较优化前后的性能和正确性。

章节串法：

- 环境章节：检查 NPU、CANN、`torch_npu`，记录版本。
- 推理章节：先跑通 Qwen 小模型，再根据机器条件记录 7B、14B 或更大模型结果。
- 微调章节：完成 LoRA 微调，验证模型输出变化。
- profiling 章节：定位推理热点，解释 ATC/OM、GE、HCCL 这类组件在问题链路中的位置。
- 算子章节：从简化版算子开始，再接入模型链路。
- 总结章节：整理性能记录、代码结构、问题清单和可复现实验表。

这个案例适合做成 hello-cann 的核心样板。它覆盖模型迁移、服务部署、性能分析、算子开发和优化验证几件事，也方便学习者整理成完整的项目记录。

### 案例二：视觉或多模态模型推理优化

这是可选拓展案例，适合放在课程后半段或第二阶段。

案例目标：

- 选择一个视觉、视频或多模态模型，在昇腾 NPU 上完成推理链路。
- 使用 profiling 分析整网耗时，定位 RMSNorm、MatMul、数据搬运、图模式或服务化相关瓶颈。
- 尝试融合算子、图模式、量化或服务化优化。
- 输出端到端优化报告，包括优化前后指标和未解决问题。

这个案例用于展示 CANN 在视觉、多模态或视频模型上的优化过程。第一阶段先预留接口和记录模板，后续再补完整实验。

- 代码可以从头运行，关键输出能证明任务完成。
